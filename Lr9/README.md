# Лабораторна робота №9
## Тема:
Дослідження прав доступу та облікових записів користувачів в операційній системі Linux.

## Мета роботи:
Ознайомитися з механізмами керування доступом у Linux, дослідити права доступу до файлів, власників файлів, роботу команд whoami, id, ls -l, chmod, chown, а також виконання команд від імені адміністратора.

## Завдання 9.1
### Умова
Напишіть програму, яка читає файл /etc/passwd за допомогою команди getent passwd, щоб дізнатись, які облікові записи визначені на вашому комп’ютері. Програма повинна визначити, чи є серед них звичайні користувачі (ідентифікатори UID повинні бути більші за 500 або 1000, залежно від вашого дистрибутива), окрім вас. 
### Компіляція і запуск
```
gcc task_9_1.c -o task_9_1
./task_9_1
```
### Результат виконання

<img width="511" height="129" alt="image" src="https://github.com/user-attachments/assets/500d32ac-2e2b-4cbe-8cd0-1aac706bbf7e" />
 
### Пояснення
У ході виконання програма використовує системну команду getent passwd, яка повертає список усіх облікових записів, визначених у системі. Отримані дані построково обробляються: із кожного запису виділяється ім’я користувача та його UID.
Програма визначає звичайних користувачів за значенням UID (як правило, від 1000). Додатково виключається поточний користувач, щоб перевірити наявність інших облікових записів.

### Висновок
Було встановлено, що система містить лише поточного користувача як звичайного. Програма коректно визначає тип користувачів за UID та дозволяє аналізувати облікові записи системи.

### Результат виконання

### Пояснення
Програма виконує системну команду sudo cat /etc/shadow. Файл /etc/shadow містить конфіденційну інформацію (зашифровані паролі користувачів) і недоступний для звичайних користувачів.
Використання sudo дозволяє тимчасово отримати адміністративні права, після чого стає можливим читання цього файлу.

### Висновок
Було підтверджено, що доступ до захищених файлів можливий лише за наявності прав адміністратора. Це є важливим елементом безпеки системи Linux.

## Завдання 9.3
### Умова
Напишіть програму, яка від імені root копіює файл, який вона перед цим створила від імені звичайного користувача. Потім вона повинна помістити копію у домашній каталог звичайного користувача. Далі, використовуючи звичайний обліковий запис, програма намагається змінити файл і зберегти зміни. Що відбудеться? Після цього програма намагається видалити цей файл за допомогою команди rm. Що відбудеться? 
### Компіляція і запуск
```
gcc task_9_3.c -o task_9_3
./task_9_3
```

### Пояснення
Програма створює файл user_file.txt від імені звичайного користувача. Далі за допомогою команди sudo створюється копія root_copy.txt, яка належить користувачу root.
У процесі виконання:
-	створюється файл user_file.txt; 
-	створюється файл root_copy.txt; 
-	файл root_copy.txt отримує права доступу, що обмежують запис для інших користувачів; 
-	при спробі змінити файл звичайний користувач отримує помилку доступу; 
-	при спробі видалити файл операція виконується успішно, оскільки права на видалення визначаються доступом до каталогу.

### Висновок
Було доведено, що права доступу до файлу та права доступу до каталогу мають різний вплив. Навіть якщо файл не можна змінити, його можна видалити за наявності відповідних прав на каталог.

### Пояснення
Було створено групу labgroup і директорію /tmp/group_dir. Для цієї директорії встановлено права 770, що дозволяє доступ лише власнику (root) та групі.

```

```
Файл успішно створився, що підтверджує правильну роботу доступу через групу.

Було продемонстровано, що групові права доступу дозволяють ефективно контролювати доступ до ресурсів. Користувач отримує доступ лише як член групи, а не індивідуально.
