# Лабораторна робота №11

## Тема:
Обробка сигналів в операційній системі Linux. Використання альтернативного стеку сигналів.

## Мета роботи:
Ознайомитися з механізмом обробки сигналів у Linux, дослідити роботу сигналів SIGINT та SIGTERM, навчитися встановлювати власний обробник сигналів за допомогою `sigaction()`, а також використати `sigaltstack()` для виконання обробника сигналу на альтернативному стеку.

## Завдання

### Варіант 8

### Умова
Напишіть програму, яка обробляє SIGINT або SIGTERM і використовує sigaltstack, щоб сигнал не завадив основному стеку.

## Опис програми

У програмі `variant8_11.c` реалізовано обробку сигналів `SIGINT` та `SIGTERM`. 
Сигнал `SIGINT` надсилається програмі при натисканні комбінації клавіш `Ctrl+C`.
Сигнал `SIGTERM` можна надіслати вручну з іншого терміналу за допомогою команди `kill -TERM`.

Головна особливість програми полягає в тому, що для обробника сигналів використовується не основний стек програми, а альтернативний стек, створений через `sigaltstack()`.

Для цього у програмі:
-	виділяється пам’ять для альтернативного стеку;
-	альтернативний стек реєструється за допомогою `sigaltstack()`;
-	для сигналів `SIGINT` і `SIGTERM` встановлюється обробник через `sigaction()`;
-	у прапорцях `sigaction` використовується `SA_ONSTACK`;
-	у самому обробнику перевіряється, чи справді він виконується на альтернативному стеку.

Якщо локальна змінна обробника знаходиться в межах альтернативного стеку, програма виводить повідомлення:

```
Signal handler is running on the alternative stack: YES
```
Це підтверджує, що sigaltstack() працює правильно.

```
gcc -Wall -Wextra variant8_11.c -o variant8_11
./variant8_11
```

## Пояснення роботи основних функцій
Функція malloc() використовується для виділення пам’яті під альтернативний стек.
Функція sigaltstack() реєструє окрему область пам’яті як альтернативний стек для обробки сигналів.
Функція sigaction() встановлює власний обробник для сигналів SIGINT і SIGTERM.
Прапорець SA_ONSTACK вказує операційній системі, що обробник сигналу потрібно запускати на альтернативному стеку.
Змінна stop_program має тип volatile sig_atomic_t, тому її можна безпечно змінювати всередині обробника сигналу. Після отримання сигналу вона змінюється на 1, і основний цикл програми завершується.
У кінці роботи програма вимикає альтернативний стек через SS_DISABLE і звільняє виділену пам’ять за допомогою free().

## Висновок
У ході лабораторної роботи було реалізовано програму для варіанту 8, яка обробляє сигнали SIGINT та SIGTERM в операційній системі Linux.
Під час тестування було перевірено два способи надсилання сигналів: натискання Ctrl+C для сигналу SIGINT та команда kill -TERM для сигналу SIGTERM.
В обох випадках програма успішно перехопила сигнал, виконала власний обробник і коректно завершила роботу. Також було підтверджено, що обробник сигналу виконується на альтернативному стеку, про що свідчить повідомлення:
```
Signal handler is running on the alternative stack: YES
```
Отже, завдання виконано правильно: програма використовує sigaltstack(), обробляє SIGINT і SIGTERM, а також не використовує основний стек для виконання обробника сигналів.
